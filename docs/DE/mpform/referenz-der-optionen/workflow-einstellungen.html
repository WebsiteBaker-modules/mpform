MPForm-Dokumentation: Workflow-Einstellungen

Schwierigkeitsgrad B 

Ist ein Folgeformular
Muss bei einseitigen Formularen sowie bei der ersten Seite eines mehrseitigen Formulars auf Nein stehen, bei allen folgenden Seiten (von mehrseitigen Formularen) auf Ja.
Folgeseite
Nachdem ein Formular fehlerfrei ausgefüllt und abgeschickt wurde, gibt es zwei Möglichkeiten. Falls keine Folgeseite angegeben wurde, bleibt der Benutzer auf der aktuellen Seite, es wird aber statt des Formulars der Text ausgegeben, der im nächsten Feld eingegeben wurde. Falls eine Folgeseite ausgewählt wurde, so wird diese angezeigt. Bei mehrseitigen Formularen muss hier die folgende Seite eingestellt werden.
Dankestext
Falls keine Folgeseite angegeben wurde, wird dieser Text statt des Formulars angezeigt, nachdem dieses fehlerfrei ausgefüllt wurde.
Datenübergabe an mpform_submissions-Tabelle
Dieser Text wird in das Feld body der mpform_submissions-Tabelle geschrieben. Er wird angezeigt, wenn Sie im Backend im Abschnitt unter den Formularfeldern eine Eintragung anklicken.
Namensendung für die Ergebnistabelle
Die Tabelle mpform_submissions ist für statistische Auswertungen überhaupt nicht geeignet. Daher speichert MPForm die Ergebnisse auch noch in einer Ergebnistabelle. Seit MPForm 1.3.1 können die Daten aus dieser Tabelle im Backend als CSV-Datei exportiert werden. Diese Tabelle enthält eine Spalte für jedes Formularfeld und eine Zeile für jedes abgeschickte Formular. Damit jedes Formular eine eigene Ergebnistabelle bekommt, haben diese unterschiedliche Namenserweiterungen. Standardmäßig wird hier die ID des Abschnitts eingetragen. Sie können diese jedoch ändern, um zum Beispiel die Ergebnisse mehrerer gleichartiger Formulare in einer Tabelle zusammenzufassen.
Wenn keine Ergebnistabelle geschrieben werden soll, weisen Sie hier den String "DISABLED zu.
